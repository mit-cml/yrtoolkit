Introduction

In this unit you will extend your knowledge of the MIT App Inventor platform, by making a the My Piano app, that will play notes like a piano!

Here are the contraints:

  • The app contains eight buttons, displayed next to each other, each depicting a piano key or note.
  • When a button is pressed, the corresponding note plays, using a Sound or Player component.
  • The string of notes played should display below the keyboard. For example, if the user press the C, D, A, B, and C keys, in that order, it should display C D A B C.
  • Pressing the Clear button should clear the display of notes played.
  • Use a procedure in the app to play each note. Call it PlayNote.

If you are up for the challenge, get coding! If you need some help getting started, proceed to the guide below.

Setup your Computer

My Piano

Designer

The goal is to have eight buttons, side by side. Each button, when pressed, will play a note. Note that the only thing included in the template are the note sound files, found in the Media section.

Add the following components to the app, and rename and set the properties as shown in the table.

  1. Drag out a HorizontalArrangement from the Layout Drawer.
    • Change the Height of the HorizontalArrangement to 50% and its Width to Fill Parent.
  2. Drag out the eight Buttons for the eight notes and drop them into the HorizontalArrangement.
    • Rename each Button according to the note it represents: CNote, DNote, ENote, FNote, GNote, ANote, BNote, HighCNote.
    • Change each Button's Height to Fill Parent and it's Width to 10%.
    • Change the Text property of each note to match the note: C, D, E, F, G, A, B, HighC.
    • Change the BackgroundColor for each note to a different color. You can use the screenshot on the Introduction tab as an example.
  3. Drag out a Label and a Button and place them below the HorizontalArrangement, and rename them NoteLabel and ClearButton respectively.
    • Delete the Text property for NotesLabel so it is blank.
    • Change the Text for ClearButton to "Clear Notes"
  4. Finally, add a Player component from the Media drawer, and rename it NotePlayer.

Code the Buttons

Now it's time to code the blocks to make your app make music!

Switch to the Blocks Editor.

Start with the CNote button. Code when CNote.Click to:

  • Play the correct sound file.
  • Display “C” in the NotesLabel.

Coding DNote is easy. You can Duplicate when CNote.Click by right-clicking on it and selecting "Duplicate" from the dropdown menu. Then change references to C and CNote to D and DNote.

Run the App

Stop with just the C and D Notes to see if your app is working so far. Start the MIT AI Companion app on your mobile device (if you are testing with a mobile device).

Then under the Connect menu, choose AI Companion.

Then on the Companion app, press the blue button and scan the QR code.

If you are using the Emulator, then choose Emulator under the Connect menu.

Try pressing the C and D Note buttons. Do they play the notes correctly?

PlayNote Procedure

The code blocks for when CNote.Click and when DNote.Click are very similar, and you have six more notes to code the same way. A good way to cut down on duplicate code and to abstract your code is by using a procedure.

Drag out a new procedure block from the Procedures drawer, and name the procedure "PlayNote".

Add an input parameter to the procedure, and call it "note".

Copy the blocks from when CNote.Click to the procedure, and change the text blocks referring to "C" to use note.

Call the PlayNote Procedure

Update when CNote.Click and when DNote.Click to call PlayNote with the appropriate note.

Add the .Click event blocks for the remaining notes, and add a call to the PlayNote procedure for each note.

Test the App

Test your app to make sure all the notes play as they should.

Then under the Connect menu, choose AI Companion.

Then on the Companion app, press the blue button and scan the QR code.

If you are using the Emulator, then choose Emulator under the Connect menu.

Update the PlayNote Procedure

You are going to update the PlayNote procedure now to display every note played, in sequence, rather than just the current note.

This is another reason to use a procedure. If you want to update something, you only have to do it in one place.

Clear Notes

The last thing to do now is code the when ClearButton.Click event. When the button is clicked, you want to clear the NotesLabel.

Test your App

And that's it! Try out your finished app. Make sure all the notes appear as you play them, and check that pressing the Clear button clears the displayed notes.
