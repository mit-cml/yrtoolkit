Two Button Game: Part 2

Introduction

How CloudDB Works

We will use CloudDB in the app to update each player’s score on the other player’s tablet by adding code to do the following. When a player clicks on their button, they’ll store their new score in CloudDB. The DataChanged event will be triggered every time that happens, and that will signal the other player to update the score on their tablet.

Changing Screen1

Open your TwoButtonGame app from Part 1, and switch to the Designer.

Drag in a second button to StartButtonArrangement and rename it StartButton2.

Change the Text of StartButton1 to “Start Player1” and change the Text of StartButton2 to “Start Player2”.

Add CloudDB

Drag in a CloudDB component from the Experimental drawer. It’s a non-visible component so it will appear below the viewer screen.

Switch back to the Blocks Editor.

Updating the Start Buttons

The first thing we have to do is update the Start buttons. StartButton1 should be mostly correct already.

Player2Button should not be enabled for Player 1. Change set Player2Button.Enabled to false by clicking on true and selecting false.

Updating the Start Buttons

Rather than rewrite all the code, we can duplicate StartButton1 and change it. Right click on when StartButton1.Click, and select Duplicate.

Click the drop down menu of StartButton1 and change it to StartButton2.

Updating the Start Buttons

Change set Player1.Enabled to false

And change set Player2.Enabled to true.

Saving to CloudDB

Every time a player scores a point, we will save the new score in CloudDB.

Update when Player1Button.Click to include a call CloudDB.StoreValue block.

The tag is “Player1Score” and the valueToStore is Player1Score.Text.

Saving to CloudDB

Do the same for when Player2Button.Click.

Data Changed

Every time we update a player’s score in CloudDB, that generates a when CloudDB.DataChanged event.

Drag out a when CloudDB.DataChanged block

Data Changed

We need to check which tag has been changed.

Drag out an if block, and then use the blue mutator to make an if-then-elseif block.

Drag out a when CloudDB.DataChanged block

Data Changed

Add a Logic equals block to the if block, and test if the tag is “Player1Score”.

Duplicate the equals block

Change “Player1Score” to “Player2Score” in the text block, then snap it into the else if.

Data Changed

Set the corresponding score label to the value passed to DataChanged.

Duplicate the set Player1Score.Text block and change to Player2Score.

Testing

Time to test! Because this game app sends scores to the cloud, we need to test using two devices. We can only test with one AI2 Companion, so we’ll make our project an apk file and install the app on two separate tablets.

Go to the “Build” menu at the top of the screen, and click on “App (provide QR code for .apk)”. This will start the process of building the app so that it can be installed on any tablet.

A QR code will appear in a pop-up window once the app is built. When it appears, both you and your partner should scan it using your tablets. Follow the prompts to download and install the .apk on your tablets.

Play the game against your partner on your own tablets. Does it work correctly? Do the scores update correctly?

Extend Your App

Add a Sound

Add a Sound (or Player) component. Remember that it’s non-visible so you won’t “see” it on the Viewer.

Find a sound, like this DING.WAV and download it to your computer.

Upload your sound file and set it as the component’s Source file.

In the Blocks Editor, add call Sound1.Play where you want the sound to play. Perhaps when a user clicks their button?

You can use more than one Sound component if you want different sounds to play for each Player.

Display Leaderboard

How about showing the players who is in the lead while the game is being played? And also displaying who the winner is?

Add a Label somewhere in your user interface in the Designer. Name it appropriately.

Add a Clock component from the Sensor Drawer. Name it something different from your other Clock. Because we’re using CloudDB, we want to account for any time lag between the 2 devices.

Set your new Clock’s TimerEnabled property to false (uncheck the box) and set its TimerInterval property to 5000 milliseconds (5 seconds).

Display Leaderboard

Use an if-then block to test who is leading. Depending on who is leading, set your Label appropriately.

You need to cover 3 possible conditions:

  • Player1 leads
  • Player2 leads
  • It's a tie

Might make sense to add else if and else to your if block.

To test is something is greater than, drag out an equals block from the Math drawer and change it to a greater than (>) block.

Display Leaderboard

Use Player1.Text and Player2.Text to compare which is larger.

Set the approprate text for your Label in each of your 3 conditions. Your else condition will be the tie. Use set LeaderLabel.Text.

This entire if-else-if-else block needs to be snapped in... where? (Hint: Remember where the score gets updated from before)

We can use the same if-else-if-else block for the end of the game. We can Duplicate the entire block and just change the text blocks slightly.

The new Clock component we added will wait 5 seconds for both players to finish up their game. Then the timer will go off, and that’s where the second if-else-if-else block goes.

Don’t forget to test out your new features! Listen for your sounds and look for the updates in your new Label.
